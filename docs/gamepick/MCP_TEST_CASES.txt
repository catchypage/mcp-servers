GamePick — MCP test cases (OpenAI Apps submission)
Copy one block per form field.
Connector: POST https://haos.pro/api/mcp (Host: haos.pro) or POST https://haos.pro/api/mcp/gamepick
No OAuth. Public tools: find_game (text search only), browse_top_games.

================================================================================
Positive test cases (5)
================================================================================

--------------------------------------------------------------------------------
Test Case 1 — Search by game title
--------------------------------------------------------------------------------

Scenario
User searches using a concrete game name; results appear in the widget.

User prompt
Search for Elden Ring and show me matching games.

Tool triggered
find_game (query describing the title, e.g. "Elden Ring")

Expected output
tools/call succeeds with mode results (or detail if the catalog returns a single strong match). Widget lists titles/posters; user can open one for detail.

--------------------------------------------------------------------------------
Test Case 2 — Search by genre / style (natural language)
--------------------------------------------------------------------------------

Scenario
User asks for games in a genre or style without naming one specific title.

User prompt
I want strategy games — show me what is out there.

Tool triggered
find_game (query including strategy or equivalent wording)

Expected output
Structured content reflects the search (mode search or results). Widget shows matches or a clear empty state with message.

--------------------------------------------------------------------------------
Test Case 3 — Top-rated discovery (no genre filter)
--------------------------------------------------------------------------------

Scenario
User wants highly rated picks from the catalog across genres (“surprise me”, explore the chart).

User prompt
Surprise me with some highly rated games worth trying — any genre is fine.

Tool triggered
browse_top_games (genre omitted or all genres; offset: 0)

Expected output
success true, mode results, results array from ranked catalog data, message summarizes what is shown. Read-only.

--------------------------------------------------------------------------------
Test Case 4 — Top-rated by genre + pagination
--------------------------------------------------------------------------------

Scenario
User asks for ranked games in one genre; later they can ask for “more” (next page).

User prompt
Show me the first page of top-rated RPGs.

Tool triggered
browse_top_games (genre: rpg or equivalent slug, offset: 0)

Expected output
success true, results array, genre filter applied, message mentions first title(s). Read-only.

--------------------------------------------------------------------------------
Test Case 5 — Open widget without query (manual search)
--------------------------------------------------------------------------------

Scenario
User opens the game UI to search or browse themselves.

User prompt
Open game search — I will type what I want myself.

Tool triggered
find_game (query empty or omitted)

Expected output
success true, mode search, helpful message; widget shows search UI and links to top/browse as designed.
