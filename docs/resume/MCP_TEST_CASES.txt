Resume Booster — MCP test cases (OpenAI Apps submission)
Copy one block per form field where applicable.

Positive test cases (5)
================================================================================

--------------------------------------------------------------------------------
Test Case 1 — Create from scratch
--------------------------------------------------------------------------------

Scenario
User starts a new resume without asking for a pre-filled role.

User prompt
I need a brand-new resume from scratch. Open Resume Booster and start the builder — I will fill everything myself.

Tool
open_resume_builder

Expected
Widget opens in create flow; user can pick a style and fill the form.

--------------------------------------------------------------------------------
Test Case 2 — Pre-filled job title
--------------------------------------------------------------------------------

Scenario
User states a target role and opens the app.

User prompt
I need a resume for a senior backend engineer position. Open Resume Booster with that role filled in.

Tool
open_resume_builder

Expected
Widget opens with the job title available for the resume (aligned with tool args from the model).

--------------------------------------------------------------------------------
Test Case 3 — Vacancy tailoring
--------------------------------------------------------------------------------

Scenario
User provides job posting text and wants tailoring.

User prompt
Tailor my resume to this posting using Resume Booster. Job description: We are looking for a React developer with 3+ years of experience building SPAs. Strong TypeScript required.

Tool
open_resume_builder

Expected
Widget opens in vacancy tailoring flow with the posting context available.

--------------------------------------------------------------------------------
Test Case 4 — Improve resume
--------------------------------------------------------------------------------

Scenario
User asks to improve wording and tone.

User prompt
Open Resume Booster in improve mode. Make my resume more concise and use stronger action verbs.

Tool
open_resume_builder

Expected
Widget opens in improve flow so the user can paste or edit their resume with that feedback.

--------------------------------------------------------------------------------
Test Case 5 — Named style + role
--------------------------------------------------------------------------------

Scenario
User asks for executive look and a specific title.

User prompt
Open Resume Booster: executive-style resume for VP of Engineering.

Tool
open_resume_builder

Expected
Widget opens with executive style and VP of Engineering reflected in the builder (aligned with tool args from the model).
