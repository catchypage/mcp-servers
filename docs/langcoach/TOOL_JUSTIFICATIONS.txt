open_lang_coach
================================================================================

Read Only: Yes
Describe why Read Only is set to True
The handler only returns structured content (message and optional topic). It does not write to our database or call external services; it has no server-side mutating side effects beyond the JSON response.

Open World: No
Describe why Open World is set to False
Inputs are an optional topic string; the implementation does not query arbitrary external systems or the open web. ChatGPT loads the widget via the standard MCP resource URL.

Destructive: No
Describe why Destructive is set to False
No deletion or destructive operations.

Idempotent: Yes
Describe why Idempotent is set to True
Repeated calls do not accumulate extra persistence on our side for this tool; same arguments yield the same structured response shape.

================================================================================
get_user_info
================================================================================

Read Only: No
Describe why Read Only is set to False
The handler loads users and langcoach_profiles and uses get-or-create: if no profile row exists yet, it inserts a minimal langcoach_profiles row so the widget can render. That insert is a write (not read-only), even though placement and preferences are not arbitrarily updated here.

Open World: No
Describe why Open World is set to False
Data comes from our application database (authenticated user + profile). That is bounded app state, not open-ended browsing of external networks.

Destructive: No
Describe why Destructive is set to False
No account deletion or destructive wipes; optional first-row insert uses fixed defaults only.

================================================================================
start_placement_test
================================================================================

Read Only: Yes
Describe why Read Only is set to True
Returns shuffled placement questions for the client without persisting server state for this call. No user answers are written; no database mutation for the test payload itself.

Open World: No
Describe why Open World is set to False
Inputs are bounded (optional locale); output is from a fixed internal question bank. The operation does not target arbitrary external systems or open-ended web content.

Destructive: No
Describe why Destructive is set to False
Does not delete or overwrite user data.

================================================================================
submit_placement_test
================================================================================

Read Only: No
Describe why Read Only is set to False
Scores answers and writes the placement result (level, score, percentage, date) to langcoach_profiles for the user. That is a write to stored user data.

Open World: No
Describe why Open World is set to False
Inputs are constrained to question IDs and option indices from the current test; outputs are scoring results. No arbitrary open-web or unbounded external payloads.

Destructive: No
Describe why Destructive is set to False
Updates or replaces placement fields on the user’s profile; it does not delete the account or irreversibly remove user records.

================================================================================
update_langcoach_profile
================================================================================

Read Only: No
Describe why Read Only is set to False
Updates stored preferences (theme, ui_locale) in langcoach_profiles for the authenticated user.

Open World: No
Describe why Open World is set to False
Only accepts bounded theme and locale values; no broad external or open-ended inputs.

Destructive: No
Describe why Destructive is set to False
Preference updates only; no destructive deletion of user data.
